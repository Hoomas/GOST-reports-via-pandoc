# Руководство для AI: как писать .md-отчёты под pandoc-шаблон ГОСТ ТПУ

Документ адресован LLM, которой нужно сгенерировать `.md`-файл, собираемый через `C:\Reports__pandoc_template\pandoc\build.ps1` в PDF по ГОСТу ТПУ (Times New Roman 14pt, поля 30/15/20/20мм, интервал 1.5, нумерация разделов с новой страницы).

---

## 1. Команда сборки

```powershell
C:\Reports__pandoc_template\pandoc\build.ps1 <путь_к_md_файлу>
```

PDF появится рядом с `.md`. Все стили, фильтры, библиография подключены через `gost_report.yaml` — вручную ничего подключать не нужно.

## 2. Структура `.md`-файла

### 2.1. YAML-frontmatter (шапка)

Должна стоять в самом начале файла между двумя `---`. Ниже — все поддерживаемые поля.

#### Минимум для лабораторной/курсовой с титульником

```yaml
---
doc-type: "Лабораторная работа"       # или "Курсовая работа", "РГР", "Отчёт по практике"
work-number: "3"                       # № работы, опционально; если есть — выводится "№ 3"
subject: "Теоретическая механика"      # дисциплина (обязательно при doc-type)
title: "Тема работы"                   # тема (обязательно при doc-type)
teacher-position: "доцент, к.т.н."     # должность принимающего, опционально
teacher: "Иванов И. И."                # ФИО принимающего (обязательно)
date: "2026"                           # год или полная дата, опционально; дефолт = текущий год
author: "Макаров С. А."                # один автор; дефолт = "Макаров С. А."
group: "8Е31"                          # дефолт = "8Е31"
lang: ru
toc: true                              # содержание (обязательно для отчётов)
number-sections: true                  # нумерация разделов
---
```

#### Несколько исполнителей (заменяет `author:`)

```yaml
authors:
  - name: "Макаров С. А."
  - name: "Орлов Д. А."
  - name: "Пасичник К. И."
```

Если есть `authors:` — выводится «Выполнили студенты группы …» и список ФИО с линиями подписи. Если нет — выводится «Выполнил студент группы …» с единичным `author`.

#### Документ БЕЗ титульника (например, реферат, конспект)

Просто не пиши `doc-type`. Тогда вместо титульного листа будет центрированный жирный `title`.

#### Библиография

`bibliography:` и `csl:` уже заданы в `gost_report.yaml`. Если нужна своя — переопредели:

```yaml
bibliography: ./my_sources.bib
csl: ./my_style.csl
```

### 2.2. Тело документа

#### Заголовки

- `# Заголовок раздела` → новая страница, **по центру**, нумеруется (1, 2, 3…).
- `## Подраздел` → с абзацного отступа (1.25см), нумерация 1.1, 1.2…
- `### Подподраздел` → аналогично, 1.1.1…
- `# Введение {.unnumbered}` → без номера, по центру (для ЦЕЛИ РАБОТЫ / ВВЕДЕНИЯ / ВЫВОДОВ / СПИСКА ЛИТЕРАТУРЫ).

КАЖДЫЙ `# Заголовок` начинает НОВУЮ СТРАНИЦУ (через Lua-фильтр `page-break-sections.lua`, который вставляет `\clearpage\phantomsection` перед каждым h1 кроме первого — это нужно, чтобы ссылки из оглавления вели точно на нужную страницу).

#### Абзацы

Просто пишешь текст. Pandoc сам делает абзацы по пустой строке. Первая строка каждого абзаца получит отступ 1.25см автоматически.

#### Списки

```markdown
- первый пункт со строчной буквы;
- второй пункт;
- третий пункт.
```

Маркер — тире (`–`). Первая строка пункта сдвинута на абзацный отступ.

```markdown
1. Первый пункт с большой буквы;
2. Второй пункт;
3. Третий пункт.
```

#### Таблицы (важно: Lua-фильтр уже подключён)

```markdown
: Подпись таблицы {#tbl:my-label}

| Колонка 1 | Колонка 2 | Колонка 3 |
|-----------|-----------|-----------|
| значение  | значение  | значение  |
| значение  | значение  | значение  |
```

- Подпись пишется ДО таблицы строкой `: Текст {#tbl:label}`.
- Метка `{#tbl:label}` позволяет ссылаться через `@tbl:label` или `[@tbl:label]` → даст «табл. N».
- Таблицы рендерятся через `xltabular` — равные X-колонки, рамка по всем сторонам, шапка повторяется на каждой странице при разрыве, длинные слова переносятся внутри ячеек.
- Не используй HTML-таблицы или сложные multi-row/multi-col — фильтр их не поддерживает.

#### Рисунки

```markdown
![Подпись рисунка](path/to/image.png){#fig:my-label}
```

- Подпись снизу по центру, формат «Рисунок N – Подпись».
- Ссылка: `@fig:my-label` → «рис. N».
- Картинка вставится точно в этом месте (`[H]`, не плавает).
- Размер: по умолчанию = ширина текста с сохранением пропорций. Можно явно: `{width=80%}` или `{width=10cm}`.

#### Формулы

Inline: `$E = mc^2$`.

Выключная нумерованная:

```latex
\begin{equation}
T = 2\pi\sqrt{\frac{L}{g}}.
\end{equation}
```

С меткой и crossref:

```latex
\begin{equation}\label{eq:period}
T = 2\pi\sqrt{\frac{L}{g}}.
\end{equation}
```

Ссылка: `\eqref{eq:period}` или через pandoc-crossref `$$...$${#eq:period}` + `@eq:period`.

Системы уравнений — окружение `cases` с расширенным интервалом 1.8 (настроено в шаблоне).

Весь текст на кириллице должен быть обернут в окрудение \text{}

```latex
\text{Вот тут важный текст}
```

#### Код

```markdown
```python
def hello():
    print("Hello")
```

```
- Times New Roman 14pt, нумерация строк слева, рамка вокруг блока.
- Длинные строки переносятся.

#### Цитирование литературы

В тексте: `[@key2024]` → `[1]`. Несколько: `[@a2020; @b2021]` → `[1, 2]`.

Источники — в `.bib` файле (см. `bibliography:` в YAML). В конце документа обязательно:

```markdown
# Список литературы {.unnumbered}

::: {#refs}
:::
```

Div `::: {#refs} :::` — это точка, куда pandoc подставит сгенерированную библиографию. Это позволяет ставить список литературы **до приложений**, а не в самый конец документа.

#### Приложения

После списка литературы используй команду `\app{Буква}{Название}`:

```markdown
\app{А}{Схема установки}

Здесь содержимое приложения А — рисунки, таблицы, текст.

\app{Б}{Дополнительные расчёты}

Содержимое приложения Б.
```

Каждое приложение автоматически начинается с новой страницы. По центру выводится «Приложение А» (жирным) и под ним название.

#### Центрирование произвольного текста

```latex
\begin{center}
Этот текст будет по центру страницы.
\end{center}
```

#### Перекрёстные ссылки (pandoc-crossref)

- На таблицу: `@tbl:label` → «табл. N», `[@tbl:label]` → «(табл. N)».
- На рисунок: `@fig:label`.
- На формулу: `@eq:label`.
- На раздел: `@sec:label` (метку ставь в заголовке: `# Раздел {#sec:label}`).

## 3. Стандартная структура отчёта

```markdown
---
doc-type: "Лабораторная работа"
work-number: "3"
subject: "Дисциплина"
title: "Тема"
teacher-position: "доцент, к.т.н."
teacher: "Иванов И. И."
authors:
  - name: "Макаров С. А."
date: "2026"
lang: ru
toc: true
number-sections: true
---

# Цель работы {.unnumbered}

Текст.

# Теоретическая часть

Текст с формулами, цитатами [@source1].

## Подраздел

Текст.

# Экспериментальная часть

Описание установки, рисунок:

![Схема установки](images/setup.png){#fig:setup}

См. @fig:setup.

# Результаты

Таблица:

: Результаты измерений {#tbl:results}

| Параметр | Значение | Единица |
|----------|----------|---------|
| ...      | ...      | ...     |

См. @tbl:results.

# Выводы {.unnumbered}

Краткие выводы.

# Список литературы {.unnumbered}

::: {#refs}
:::

\app{А}{Название приложения А}

Содержимое приложения А.
```

## 4. Что НЕЛЬЗЯ / нежелательно

- **HTML-теги** (`<br>`, `<div>`, `<table>`) — игнорируются или ломают рендер.
- **Сложные таблицы** с объединением ячеек, multirow/multicolumn в Markdown — Lua-фильтр их не понимает; всё развалится. Если нужно — пиши `\begin{table} ... \end{table}` сырым LaTeX.
- **Markdown-таблицы без подписи** — выглядят сиротливо; всегда добавляй `: Подпись {#tbl:label}` перед.
- **Эмодзи / экзотические Unicode-символы** — XeLaTeX может упасть, если глифа нет в Times New Roman.
- **Картинки без метки** (`#fig:`) — нельзя сослаться.
- **Самодельный заголовок `\section*{Список литературы}` в LaTeX-блоке** — даст дубль. Только `# Список литературы {.unnumbered}` markdown'ом + `::: {#refs} :::`.
- **`# Список литературы` БЕЗ `::: {#refs} :::`** — библиография уйдёт в самый конец документа (после приложений). Всегда ставь div явно.

- Автор по умолчанию: **Макаров С. А.**

## 6. Типичные ошибки сборки

| Симптом                                | Причина                                   | Лечение                                  |
| -------------------------------------- | ----------------------------------------- | ---------------------------------------- |
| `permission denied` на `.pdf`          | PDF открыт в просмотрщике                 | Закрыть просмотрщик и пересобрать        |
| Длинные слова вылезают за поле         | Сильно длинное слово без переносов        | Добавить мягкий перенос `\-` или разбить |
| Цитата `[@key]` показывается как `[?]` | Ключа нет в `.bib`                        | Добавить запись или исправить ключ       |
| Кириллица в формулах ломается          | `unicode-math` не любит русский в `$...$` | Использовать `\text{рус}` внутри формулы |
| Таблица не нумеруется                  | Нет подписи `: ...` перед                 | Добавить подпись                         |
