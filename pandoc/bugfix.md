# Задача: исправить баги в pandoc-шаблоне ВКР/отчётов (ГОСТ)

## Контекст

У меня есть pandoc-пайплайн для сборки учебных работ из Markdown в PDF через XeLaTeX. Сборка работает в двух режимах: через Zettlr (GUI) и через CLI pandoc (скрипт `build.ps1`). Оба используют одну и ту же конфигурацию.

Расположение файлов:
- `C:\Reports__pandoc_template\pandoc\gost_report.yaml` — параметры pandoc (defaults)
- `C:\Reports__pandoc_template\pandoc\template.tex` — LaTeX-шаблон под XeLaTeX
- `C:\Reports__pandoc_template\pandoc\filters\gost-tables.lua` — Lua-фильтр оформления таблиц по ГОСТ
- `C:\Reports__pandoc_template\pandoc\bibliography\sources.bib` — библиография (.bib)
- `C:\Reports__pandoc_template\pandoc\bibliography\gost-r-7-0-5-2008-numeric.csl` — стиль ГОСТ
- `C:\Reports__pandoc_template\pandoc\build.ps1` — скрипт сборки

ОС: Windows. Запуск сборки через PowerShell:
```powershell
C:\Reports__pandoc_template\pandoc\build.ps1 <путь_к_md_файлу>
```

Скрипт сам обрабатывает кодировку UTF-8, переходит в директорию документа, собирает PDF рядом с .md. Возвращает код выхода pandoc.

В конфигурации есть несколько известных проблем, которые нужно последовательно устранить. Не вноси других изменений, не относящихся к этому списку. После каждой правки прогоняй тестовую сборку и убеждайся, что она проходит без LaTeX-ошибок, прежде чем переходить к следующей.

## Правки

Выполняй строго в этом порядке. После каждой — пересобирай PDF и проверяй, что нет регрессий. Игнорируй косметические warning'и про:

### Правка 1: Опечатка в gost_report.yaml

В файле `C:\Reports__pandoc_template\pandoc\gost_report.yaml`, в самой первой строке найди:
```yaml
reader: markdown+smartю
```
Заменить на:
```yaml
reader: markdown+smart
```
Лишняя кириллическая «ю» в конце — опечатка.

**Проверка:** сборка проходит без ошибок чтения defaults.

### Правка 2: Убрать дубли и мусорные поля в gost_report.yaml

В `gost_report.yaml`:

- В блоке `variables:` удалить строки `biblatex: false` и `natbib: false` — это значения по умолчанию.
- В блоке `variables:` удалить строку `numbersections: true` — дублирует `number-sections: true` на верхнем уровне. Оставь только верхнеуровневый.
- В блоке `metadata:` удалить строку `title: "Выпускная квалификационная работа"` — этот title будет перекрывать title из YAML-frontmatter каждой работы.

**Проверка:** сборка проходит; title в PDF берётся из тестового файла, а не из defaults.

### Правка 3: Удалить запрет переносов в template.tex

Это главная причина проблемы с длинными словами, вылезающими за границы ячеек таблиц.

В файле `template.tex` найди и удали (или закомментируй) строки:
```latex
\hyphenpenalty=10000
\exhyphenpenalty=10000
\sloppy
```
Все три. `\sloppy` тоже убираем — он даёт «дыры» между словами.

**Проверка:** сборка проходит. Длинные слова в обычных абзацах теперь переносятся со знаком переноса.

### Правка 4: Разрешить перенос первого слова в ячейках таблиц

В `template.tex` после `\usepackage{xltabular}` и определений `newcolumntype` (в секции «6. ТАБЛИЦЫ») добавь:

```latex
% Разрешить перенос первого слова в X-колонках
% Магическое \hspace{0pt} позволяет TeX переносить первое слово в ячейке
\renewcommand\tabularxcolumn[1]{>{\justifying\arraybackslash\hspace{0pt}}p{#1}}
```

**Проверка:** в тестовой таблице длинные слова теперь переносятся внутри ячеек и не вылезают за границы.

### Правка 5: Линии при разрыве таблицы между страницами

В файле `gost-tables.lua`, в функции `Table(el)`, найди блок «-- 4. Заголовок» и строчку `\hline` перед ним. Текущая структура:

```lua
  table.insert(tex, "\\hline\n")

  -- 4. Заголовок
  if el.head and el.head.rows then
    for _, row in ipairs(el.head.rows) do
       table.insert(tex, row_tex(row, num_columns))
    end
    table.insert(tex, "\\endfirsthead\n")
    for _, row in ipairs(el.head.rows) do
       table.insert(tex, row_tex(row, num_columns))
    end
    table.insert(tex, "\\endhead\n")
  end
```

Заменить на:

```lua
  -- 4. Заголовок таблицы
  if el.head and el.head.rows then
    -- firsthead: рисуется на ПЕРВОЙ странице таблицы
    table.insert(tex, "\\hline\n")
    for _, row in ipairs(el.head.rows) do
       table.insert(tex, row_tex(row, num_columns))
    end
    table.insert(tex, "\\endfirsthead\n")

    -- head: рисуется в начале КАЖДОЙ последующей страницы при разрыве
    table.insert(tex, "\\multicolumn{" .. num_columns .. "}{l}{\\textit{Продолжение таблицы}} \\\\\n")
    table.insert(tex, "\\hline\n")
    for _, row in ipairs(el.head.rows) do
       table.insert(tex, row_tex(row, num_columns))
    end
    table.insert(tex, "\\endhead\n")
  end

  -- 4.5. Подвал при разрыве страницы (foot) и финальный подвал (lastfoot)
  table.insert(tex, "\\hline\n")
  table.insert(tex, "\\multicolumn{" .. num_columns .. "}{r}{\\textit{Продолжение на следующей странице}} \\\\\n")
  table.insert(tex, "\\endfoot\n")
  table.insert(tex, "\\hline\n")
  table.insert(tex, "\\endlastfoot\n")
```

Обрати внимание: отдельная `\hline\n` перед заголовком убрана, потому что теперь она внутри `\endfirsthead`.

**Проверка:** собери документ с большой таблицей (она есть в test-fixes.md, занимает 2+ страницы). В PDF проверь визуально:
- На первой странице вверху таблицы — горизонтальная линия и шапка.
- Внизу первой страницы (перед разрывом) — горизонтальная линия и текст «Продолжение на следующей странице».
- На второй странице вверху — текст «Продолжение таблицы», линия и повтор шапки.
- В конце таблицы — закрывающая горизонтальная линия.

Если что-то отсутствует — фикс не сработал, разбирайся.

### Правка 6: Убрать дубль заголовка «Список литературы»

В `template.tex` в окружениях `CSLReferences` и `thebibliography` зашит `\section*{Список литературы}`. При этом pandoc сам вставляет заголовок секции, если в Markdown есть `# Список литературы`. Это даёт два одинаковых заголовка.

Найди:
```latex
\newenvironment{CSLReferences}[2]
  {\section*{Список литературы}
   \begin{description}[font=\normalfont, style=multiline, leftmargin=0pt]}
  {\end{description}}
```

Заменить на:
```latex
\newenvironment{CSLReferences}[2]
  {\begin{description}[font=\normalfont, style=multiline, leftmargin=0pt]}
  {\end{description}}
```

И:
```latex
\renewenvironment{thebibliography}[1]
  {\section*{Список литературы}
   \begin{description}[font=\normalfont, style=multiline, leftmargin=0pt]}
  {\end{description}}
```

Заменить на:
```latex
\renewenvironment{thebibliography}[1]
  {\begin{description}[font=\normalfont, style=multiline, leftmargin=0pt]}
  {\end{description}}
```

Заголовок теперь задаётся в самом .md через `# Список литературы` перед местом, где pandoc подставит список.

**Проверка:** в test-fixes.md есть `# Список литературы` и хотя бы одна цитата `[@test2024]` плюс соответствующая запись в `sources.bib`. В PDF — ровно один заголовок «Список литературы», не два.
